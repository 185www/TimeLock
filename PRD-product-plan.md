# 时间锁 (TimeLock) — 产品策划文档

## 1. 产品概述

### 1.1 背景
参考何同学「时间锁」概念，做一个真正好用的、安全的、持续维护的防沉迷工具。  
核心哲学：**你设定时间，手机在这段时间内彻底变为「砖头」，让你强制专注于现实世界。**

### 1.2 目标用户
- 学生党：备考、自习时管不住自己刷手机
- 上班族：需要深度工作、远离社交媒体
- 任何有手机成瘾倾向、想戒断的人

### 1.3 核心理念与原有 App 的区别

| 维度 | 原有 App | 本产品 |
|------|----------|--------|
| 可靠性 | 可被强行退出/卸载/杀进程 | **系统级防护，无法绕过** |
| 更新维护 | 多年不更新 | 持续维护，开源社区驱动 |
| 权限 | 普通 APP 权限 | 利用无障碍 + 设备管理员 + 屏幕固定 |
| 代码 | 闭源 | **完全开源，可审计，无后门** |
| 自定义 | 固定模式 | 灵活的时间/白名单/紧急策略 |

## 2. 核心功能

### 2.1 锁机模式（核心）
- 用户设定锁定时长（1 分钟 ~ 24 小时）
- 倒计时期间：
  - 屏幕锁定在 App 界面，无法退出（利用 Android **屏幕固定 / Screen Pinning**）
  - 返回键、Home 键、多任务键全部屏蔽
  - 通知栏折叠，不显示内容
  - 无法关机（？硬件限制，但可监听关机广播，重启后自动恢复锁定）
- 倒计时结束后自动解锁，恢复正常使用

### 2.2 白名单
- 紧急联系人电话可接听（使用锁屏界面拨号器）
- 白名单 App（如计算器、地图）可在锁定时使用
- 紧急一键解铃（预留机制，但会记录日志，用户需为滥用负责）

### 2.3 强制模式（严格版）
- 锁机期间**无法卸载 App**
- 利用设备管理员权限禁止停用本 App
- 锁机期间无法重启绕开（重启后自动恢复）
- 锁定期间进入飞行模式？可选项

### 2.4 统计与激励
- 每日/每周/每月锁机时长统计
- 连续锁机成就系统
- 番茄钟模式（25+5 标准工作法）
- 专注排行榜（可选，本地 or 可选云同步）

## 3. 技术架构

### 3.1 技术选型

| 层次 | 方案 |
|------|------|
| 语言 | **Kotlin**（主） + Compose (Jetpack Compose) |
| 最低 SDK | Android 8.0 (API 26) |
| 目标 SDK | Android 14 (API 34) |
| 架构 | MVVM + Repository Pattern |
| 依赖注入 | Hilt |
| 异步 | Kotlin Coroutines + Flow |
| 构建 | Gradle Kotlin DSL |
| CI/CD | GitHub Actions |
| 签名 | 使用 GitHub Actions + 签名的 release APK/AAB |

### 3.2 核心权限策略

| 权限 | 用途 | 可选/必须 |
|------|------|-----------|
| `Screen Pinning` | 锁定屏幕，防止退出 | **必须** |
| `Device Admin` | 防止卸载，锁屏控制 | 可选（强制模式） |
| `Usage Stats` | 统计使用时长 | 可选 |
| `Notification Listener` | 屏蔽通知内容 | 可选 |
| `Overlay` | 锁定时的浮层提示 | 可选 |
| `Phone` | 紧急呼叫 | 可选 |

### 3.3 安全设计（重中之重）

```
┌─────────────────────────────────────────────┐
│           TimeLock 安全架构                   │
├─────────────────────────────────────────────┤
│ 1. 屏幕固定 (Screen Pinning) — 系统级锁定      │
│    用户无法退出，只有输入密码/生物识别可解锁    │
│ 2. 设备管理员权限 — 防止应用被卸载/强制停止     │
│ 3. 服务守护 — Foreground Service + 高优先级   │
│    通知，系统难以杀死的进程                   │
│ 4. 重启恢复 — BOOT_COMPLETED 广播自动恢复锁定 │
│ 5. 防绕过检测 — 检测无障碍/ADB/开发者选项状态  │
│ 6. 紧急机制 — 安全密码+日志，防滥用            │
└─────────────────────────────────────────────┘
```

### 3.4 绕过防护
- 检测开发者选项是否开启 → 提示关闭
- 检测 ADB 是否连接 → 锁定时禁止 ADB 命令
- 检测无障碍服务是否被关闭 → 弹窗提醒
- 检测是否在安全模式启动 → 提示风险
- 锁屏期间禁止下拉通知栏、禁止快捷设置

## 4. 产品路线图

### Phase 1 — MVP (1-2 周)
- 基础锁机功能：设定时间 → 锁屏 → 倒计时 → 解锁
- Screen Pinning 实现锁定
- Foreground Service 保障存活
- 基本的 UI（Compose）
- GitHub Actions 构建 APK + Release

### Phase 2 — 完善 (2-4 周)
- 设备管理员权限集成
- 白名单 App/联系人
- 统计图表
- 重启恢复机制
- English / 中文双语

### Phase 3 — 进阶 (4-8 周)
- 番茄钟模式
- 成就系统
- 云同步（可选）
- 小组件（Widget）
- Material You 动态主题

### Phase 4 — 长期
- 开源社区贡献者指南
- 安全审计
- 国际化多语言
- iOS 版本（不同架构，但理念复用）

## 5. 隐私与安全承诺

- **绝对不开后门**：App 完全开源，任何安全研究员可审计代码
- **无网络权限**（可选）：核心功能完全不联网，本地运行
- **不上传任何数据**：所有统计数据仅存储在本地
- **紧急机制透明**：紧急解锁的逻辑完全开源，无隐藏通道
- **签名可验证**：Release 构建通过 GitHub Actions，签名哈希公开

## 7. 发布策略

- **GitHub Releases**: 每次 tag 自动构建 APK + AAB，发布到 Releases
- **F-Droid**: 上架 F-Droid 开源应用市场
- **Google Play**: 可选上架（需遵循 Google Play 政策）

## 8. 成功指标

- GitHub Stars >= 100（社区认可）
- 日活用户（DAU）的锁机完成率 > 90%（说明防绕过有效）
- App 崩溃率 < 0.1%
- 平均锁机时长 > 45 分钟（说明真的有帮到用户）
